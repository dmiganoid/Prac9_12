# Project instructions

Проект выполняется для учебного практического цикла по теме:
"Микросервис для генерации PDF-отчётов по данным из БД".

Основная цель текущего этапа — практика №2: реализация MVP микросервисной системы.

## Общие требования

- Использовать Python 3.12.
- Использовать FastAPI для HTTP-сервисов.
- Использовать PostgreSQL для хранения бизнес-данных и заявок на отчёты.
- Использовать Redis Streams как очередь задач.
- Использовать MinIO как S3-совместимое хранилище PDF-файлов.
- Использовать ReportLab для генерации PDF, чтобы избежать сложных системных зависимостей WeasyPrint.
- Каждый микросервис должен иметь Dockerfile.
- В корне проекта должен быть docker-compose.yml для запуска всей системы.
- Добавить автотесты pytest.
- Добавить health endpoints:
  - GET /health/live
  - GET /health/ready
- Добавить endpoint /metrics с Prometheus-метриками.
- Не реализовывать Kubernetes-манифесты в рамках этой задачи. Kubernetes будет сделан в practice3.
- Не реализовывать полноценную авторизацию. Это учебный MVP.

## Стиль реализации

- Не усложнять проект лишними абстракциями.
- Добавлять комментарии только там, где они действительно помогают понять логику.
- Все настройки брать из переменных окружения.
- Добавить .env.example.
- API должен быть документирован через FastAPI Swagger.
